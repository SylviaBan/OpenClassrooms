Analyse de stabilité¶

Enjeux¶

Dans le but d'tablir un contrat de maintenance de l'algorithme de segmentation client, nous devons tester sa stabilit dans le temps et voir, par exemple, quel moment les clients changent de cluster.

Pour cela, nous devons recalculer toutes les features en fonction d'une priode donne.

En d'autres termes, il s'agit de faire une simulation pour dterminer la frquence ncessaire de mise jour du modle de segmentation. Pour cela, nous allons comparer les segments dj fournis par notre algorithme avec ceux du futur. C'est l'indice de Rand qui nous permettra d'valuer leur similarit.

Indice de Rand¶

L'indice de Rand est utilis pour mesurer la similarit des points de donnes prsents dans les clusters. Plus prcisment, il s'agit de prendre en compte toutes les paires d'observations et compter les paires qui sont assignes aux mmes clusters (ou des clusters diffrents).

Ainsi, cette mesure doit tre aussi leve que possible, sinon nous pouvons supposer que les points de donnes sont attribus au hasard.

Méthodologie¶

Nous souhaitons donc savoir si notre algorithme fournira les mmes segmentations dans le futur. Afin de simuler le futur, nous allons :

  1. Mettre en place plusieurs jeux de donnes :

    • Dataframe de rfrence :
      • B0 la date T0
    • Dataframes artificiels du futur :
      • B1 - B0 plus 1 mois (T1.merge(T0))
      • B2 - B0 plus 2 mois (T2.merge(T0))
      • B3 - B0 plus 3 mois (T3.merge(T0)) etc.
  1. Ensuite nous allons effectuer un clustering sur les bases artificielles :
    • C1_cluster.fit(B1_df)
    • C1_cluster.fit(B2_df)
    • C1_cluster.fit(B3_df) etc.
  1. Ensuite, nous allons gnrer une liste regroupant les labels (prdictions) attribus :
    • cluster_labels_B1
    • cluster_labels_B2
    • cluster_labels_B3 etc.
  1. Finalement, nous allons comparer la segmentation entre C0 et Cx grce l'indice de Rand ajust :
    • adjusted_rand_score(cluster_labels_t0, cluster_labels_B1)
    • adjusted_rand_score(cluster_labels_t0, cluster_labels_B2)
    • adjusted_rand_score(cluster_labels_t0, cluster_labels_B3) etc.
  1. Affichage de l'volution de l'indice de Rand ajust.

# Rcence
def rec(df, start, end):
    # Dfinition du cadre temporaire
    rec = df.loc[(df['order_purchase_timestamp'] > start) & (df['order_purchase_timestamp'] < end)]
    # Calcul de la rcence
    rec = rec[['customer_unique_id', 'order_purchase_timestamp']].copy()
    rec = rec.groupby('customer_unique_id')['order_purchase_timestamp'].max().reset_index()
    rec.columns = ['customer_unique_id', 'last_purchase_timestamp']
    # Date du dernier achat max - date du jour
    rec['Rcence'] = (rec['last_purchase_timestamp'].max() - rec['last_purchase_timestamp']).dt.days
    rec.drop(columns='last_purchase_timestamp', inplace=True)
    return rec

# Frquence
def freq(df, start, end):
    # Dfinition du cadre temporaire
    freq = df.loc[(df['order_purchase_timestamp'] > start) & (df['order_purchase_timestamp'] < end)]
    # Calcul de la frquence
    freq = freq[['customer_unique_id', 'order_id']].copy()
    # Addition du nombre de commandes passes
    freq = freq.groupby('customer_unique_id')['order_id'].count().reset_index()
    freq.columns = ['customer_unique_id', 'Frquence']
    return freq

# Montant
def montant(df, start, end):
    # Dfinition du cadre temporaire
    montant = df.loc[(df['order_purchase_timestamp'] > start) & (df['order_purchase_timestamp'] < end)]
    # Calcul du montant dpens
    montant = montant[['customer_unique_id','payment_value']].copy()
    # Somme totale des paiements effectus
    montant = montant.groupby('customer_unique_id')['payment_value'].sum().reset_index()
    montant.columns = ['customer_unique_id', 'Montant']
    return montant

# Avis
def avis_mean(df, start, end):
    # Dfinition du cadre temporaire
    avis_mean = df.loc[(df['order_purchase_timestamp'] > start) & (df['order_purchase_timestamp'] < end)]
    # Calcul de la moyenne de la satisfaction client
    avis_mean = avis_mean[['customer_unique_id', 'review_score']].copy()
    avis_mean = avis_mean.groupby('customer_unique_id')['review_score'].mean().reset_index()
    avis_mean.columns = ['customer_unique_id', 'Avis moy']
    return avis_mean

Pour dterminer le moment o les clients changent de cluster, nous allons itrer le K-Means sur toute la priode avec des deltas de 1 mois et calculer le score ARI, en prenant garde bien comparer les mmes clients (ceux des 12 mois initiaux).

from sklearn.preprocessing import PowerTransformer

def clustering (df):
    df = df[['Rcence', 'Frquence', 'Montant', 'Avis moy']].copy()
    
    # Transformation des donnes pour k-means 
    # PowerTransformer est le meilleur transformer en termes de prise en compte des outliers
    transformer = PowerTransformer(method='yeo-johnson')
    df_normalized = transformer.fit_transform(df)
    
    # k-means sur donnes transformes
    kmeans = KMeans(n_clusters=5, random_state=1)
    kmeans.fit(df_normalized)
    
    # rcupration des labels
    return kmeans.labels_

Calcul du score ARI - Adjusted Rand Index

L'indice de Rand peut tre gonfl artificiellement en prdisant beaucoup de clusters : les paires de points appartenant des clusters diffrents seront nombreuses, et il y aura de grandes chances que deux points tiquets diffremment soient dans deux clusters diffrents.

L'indice de Rand ajust (ARI, pour Adjusted Rand Index) corrige cet effet en normalisant l'indice de Rand (RI).

Cet index ajust est proche de 0 pour un clustering alatoire et gal 1 uniquement quand le clustering correspond exactement la partition initiale.

A prsent nous allons calculer les scores ARI et ensuite afficher leur volution.

Conclusion¶

Sur le graphique nous observons des scores ARI obtenus sur les itrations par priode de 1 mois. Nous remarquons une forte inflexion aprs 5 mois sur les clients initiaux.

Il faudra donc prvoir la maintenance du programme de segmentation tous les 5 mois dans un premier temps puis re-tester cette stabilit temporelle au fil du temps afin de l'affiner. Il sera donc ncessaire de redfinir les segments clients chaque maintenance.
